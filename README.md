# Intelligent LoRa Gateway

**Author:** Naveed Akhtar

**Degree:** Master of Science: Communication & Information Technology (CIT)

## Project Overview
Sensors, a Gateway, and a Network/Cloud-Server typically make up a conventional LoRaWAN Network. However, in use cases like monitoring the temperature inside a truck, an interruption in external communication makes it impossible to adjust cooling set points. 

The goal of this project is to develop an Intelligent LoRa Gateway based on a Raspberry Pi to make local decisions and control systems directly. By utilizing our own gateway as a domain controller, we protect incoming data locally and eliminate the dependency on external network servers like The Things Network (TTN).

## Architecture & Data Flow
The data flow tracks how information is generated by an end device and carried through the network:
1. **Sensing:** The RN2483 LoRa Motes utilize onboard light and temperature sensors to generate data.
2. **Transmission:** The RN2483 encrypts the data and transmits it via the LoRa network. 
3. **Gateway Reception:** The Raspberry Pi-based LoRaWAN Gateway receives the radio packet and forwards it to a local UDP server.
4. **Decoding:** A local Node.js UDP server receives the payload in Base64 format. The JavaScript application decodes the Base64 data into HEX decimal, verifies the Message Integrity Code (MIC), and decrypts the physical payload into readable ASCII text.

## Hardware Used
* **Microchip RN2483 LoRa Technology Transceiver Module:** A low-power, long-range LoRa module used as the end-node (Motes).
* **Raspberry Pi 3B+:** The core single-board computer running the Raspbian OS.
* **LRWCCx-MPCIE Concentrator Card:** A micro PCIe-based LoRaWAN concentrator card based on Semtech's SX1301 or SX1308 chipset.
* **SBCPOE RPi Hat:** Provides PoE power to the Raspberry Pi and houses the concentrator card.

## Software & Dependencies
The entire decoding logic and server were written in JavaScript (Node.js) on the Raspbian operating system.

**Required NPM Libraries:**
* `crypto-js`
* `lora-packet`: Used to decode receiving Base64 data, parse the packet structure (PHYPayload), and verify the MIC.
* `node-aes-cmac`

## Installation & Setup

### 1. Gateway Configuration
1. Connect the Gateway to your local network via Ethernet (PoE) and log in via SSH.
2. Update the Raspbian OS:
   ```bash
   sudo enableWriteAccess.sh
   sudo apt-get update && sudo apt-get -y dist-upgrade
   sudo disableWriteAccess.sh
3. Copy the appropriate frequency template (e.g., EU 868MHz) to your global configuration:
   ```bash
   sudo cp /usr/share/doc/lora-pkt-fwd/conf/global_conf_PicoV1p0_EU.json /etc/lora_pkt_fwd/global_conf.json
4. Obtain your Gateway ID (sudo util_chip_id -d /dev/serial0) and update the gateway_ID and server_address (to your local IP) in /etc/lora_pkt_fwd/global_conf.json
5. Enable and start the forwarder service:
   ```bash
   sudo systemctl enable lora_pkt_fwd.service
   sudo systemctl start lora_pkt_fwd.service
### 2. Decoder Setup
1. Install Node.js (v14.x) on the Raspberry Pi:
   ```bash
   curl -sL [https://deb.nodesource.com/setup_14.x](https://deb.nodesource.com/setup_14.x) | sudo -E bash -
   sudo apt-get install -y nodejs
2. Clone this repository and install dependencies:
   ```bash
   npm install
3. Start the decoding program:
   ```bash
   node decoder.js

## Configuring the RN2483 Node
Connect the RN2483 to a PC, open the Arduino IDE Serial Monitor, and configure it for Activation by Personalization (ABP):
```Plaintext
sys reset
mac set nwkskey <Your_Network_Session_Key>
mac set appskey <Your_App_Session_Key>
mac set devaddr <Your_Device_Address>
mac set adr off
mac save
mac join abp
mac tx uncnf 1 68656c6c6f576f726c64
```
