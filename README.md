# Intelligent LoRa Gateway

[cite_start]**Author:** Naveed Akhtar [cite: 6]  
[cite_start]**Institution:** University of Bremen, Institute for Microsensors, -actuators and -systems (IMSAS) [cite: 1, 2, 14]  
[cite_start]**Degree:** Master of Science: Communication & Information Technology (CIT) [cite: 9]

## Project Overview
[cite_start]Sensors, a Gateway, and a Network/Cloud-Server typically make up a conventional LoRaWAN Network[cite: 30].
[cite_start]However, in use cases like monitoring the temperature inside a truck, an interruption in external communication makes it impossible to adjust cooling set points[cite: 43, 45]. 

[cite_start]The goal of this project is to develop an Intelligent LoRa Gateway based on a Raspberry Pi to make local decisions and control systems directly[cite: 46].
[cite_start]By utilizing our own gateway as a domain controller, we protect incoming data locally and eliminate the dependency on external network servers like The Things Network (TTN)[cite: 307, 311, 312].

## Architecture & Data Flow
[cite_start]The data flow tracks how information is generated by an end device and carried through the network[cite: 483]:
1. [cite_start]**Sensing:** The RN2483 LoRa Motes utilize onboard light and temperature sensors to generate data[cite: 466].
2. [cite_start]**Transmission:** The RN2483 encrypts the data and transmits it via the LoRa network[cite: 485]. 
3. [cite_start]**Gateway Reception:** The Raspberry Pi-based LoRaWAN Gateway receives the radio packet and forwards it to a local UDP server[cite: 485, 516].
4. [cite_start]**Decoding:** A local Node.js UDP server receives the payload in Base64 format[cite: 445].
5. [cite_start]The JavaScript application decodes the Base64 data into HEX decimal, verifies the Message Integrity Code (MIC), and decrypts the physical payload into readable ASCII text[cite: 445, 964, 1055].

## Hardware Used
* [cite_start]**Microchip RN2483 LoRa Technology Transceiver Module:** A low-power, long-range LoRa module used as the end-node (Motes)[cite: 239, 298].
* [cite_start]**Raspberry Pi 3B+:** The core single-board computer running the Raspbian OS[cite: 276, 823].
* [cite_start]**LRWCCx-MPCIE Concentrator Card:** A micro PCIe-based LoRaWAN concentrator card based on Semtech's SX1301 or SX1308 chipset[cite: 278].
* [cite_start]**SBCPOE RPi Hat:** Provides PoE power to the Raspberry Pi and houses the concentrator card[cite: 281, 283].

## Software & Dependencies
[cite_start]The entire decoding logic and server were written in JavaScript (Node.js) on the Raspbian operating system[cite: 816, 817].

**Required NPM Libraries:**
* [cite_start]`crypto-js` [cite: 1025]

## Installation & Setup

### 1. Gateway Configuration
1. [cite_start]Connect the Gateway to your local network via Ethernet (PoE) and log in via SSH[cite: 826, 831].
2. Update the Raspbian OS:
   ```bash
   sudo enableWriteAccess.sh
   sudo apt-get update && sudo apt-get -y dist-upgrade
   sudo disableWriteAccess.sh
3. Copy the appropriate frequency template (e.g., EU 868MHz) to your global configuration:
   ```bash
   sudo cp /usr/share/doc/lora-pkt-fwd/conf/global_conf_PicoV1p0_EU.json /etc/lora_pkt_fwd/global_conf.json
4. Obtain your Gateway ID (sudo util_chip_id -d /dev/serial0) and update the gateway_ID and server_address (to your local IP) in /etc/lora_pkt_fwd/global_conf.json
5. Enable and start the forwarder service:
   ```bash
   sudo systemctl enable lora_pkt_fwd.service
   sudo systemctl start lora_pkt_fwd.service
### 2. Decoder Setup
1. Install Node.js (v14.x) on the Raspberry Pi:
   ```bash
   curl -sL [https://deb.nodesource.com/setup_14.x](https://deb.nodesource.com/setup_14.x) | sudo -E bash -
   sudo apt-get install -y nodejs
2. Clone this repository and install dependencies:
   ```bash
   npm install
3. Start the decoding program:
   ```bash
   node decoder.js

## Configuring the RN2483 Node
Connect the RN2483 to a PC, open the Arduino IDE Serial Monitor, and configure it for Activation by Personalization (ABP):
```Plaintext
sys reset
mac set nwkskey <Your_Network_Session_Key>
mac set appskey <Your_App_Session_Key>
mac set devaddr <Your_Device_Address>
mac set adr off
mac save
mac join abp
mac tx uncnf 1 68656c6c6f576f726c64
```
